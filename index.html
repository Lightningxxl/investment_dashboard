1组合目标

这些是组合优化的硬约束。收益目标越高，通常需要放宽回撤、波动或 beta 暴露。

Required return%

组合调整后预期年化收益目标。当前用完整年度均值 + 可信度折扣估算。

Max drawdown%

组合最大回撤预算。当前用年度净值回撤和产品历史回撤做 proxy。

Max standard deviation%

组合波动上限。PDF 没有月度净值，当前用年度收益离散度估算。

Beta benchmark

组合 beta 和低大盘相关性都相对这个基准计算。切换后会实时重算产品分数和推荐组合。

Max beta exposure

相对当前 Beta benchmark 的组合 beta 上限。样本不足时会引入策略 beta 先验。

模型计算口径

第 1 步定义组合合格线，第 2 步定义产品可信度偏好。

1. 组合目标 用于过滤模拟组合。收益、回撤、波动和 beta 同时满足时，组合进入可行集合。 组合收益 >= Required return 组合回撤 <= Max drawdown 组合波动 <= Max stdev 组合 beta <= Max beta
2. 产品评分 每个维度先根据 PDF 数据生成 0-100 维度评分，再按左侧 0-5 重要性归一化为综合评分；单品权重上限由综合评分和组合敞口集中度共同决定。 产品综合评分 = Σ(维度评分 × 用户重要性) / Σ(用户重要性)
3. 指标计算 当前 PDF 只有年度收益，所以波动、beta 和相关性均为年度 proxy。 历史收益 = 年度收益均值 波动 = 年度收益标准差 Beta = Cov(产品, benchmark) / Var(benchmark) 相关性 = Corr(产品, benchmark)
4. 当前限制 没有月度净值、持仓行业和 AUM 数据；集中度风险、相关性矩阵和容量质量仍是 proxy。 集中度 = 策略分类 + 标签 proxy 相关性矩阵 = 年度收益 + 策略关系 AUM / 容量 = 当前未接入
当前列表 0 只 调整后收益 = 历史收益 × 可信度折扣 Beta / 相关性相对当前 benchmark 波动 = 年度收益离散度 proxy
